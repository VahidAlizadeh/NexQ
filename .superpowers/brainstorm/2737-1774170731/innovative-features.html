Innovative features for In-Person Mode

Select any combination you'd want in v1 — or mark as "later". These all build on the diarization + scenario architecture we've locked in.

1

Live Speaker Naming

When diarization detects a new speaker for the first time, show a subtle inline prompt in the transcript: "New speaker detected — name?" with a quick text input. User types "Professor Smith" and all past + future segments for that speaker ID update instantly. Much better than renaming after the meeting.

Effort: Low-Medium — UI component + speaker ID mapping
2

Real-Time Speaker Stats

A collapsible panel showing per-speaker metrics: talk time %, word count, last spoke. In a group meeting, instantly see who's dominating and who hasn't spoken. In a lecture, see the Q&A ratio. Feeds into post-meeting summary too.

Effort: Low — computed from existing transcript segments
3

Auto Topic Sections

AI periodically analyzes the rolling transcript and inserts topic dividers: "— Shifted to: Budget Discussion —". Makes long lectures and meetings scannable. Post-meeting, you get a table of contents with timestamps.

Effort: Medium — periodic LLM calls + UI for section markers
4

Live Action Item Detection

AI watches for commitments in real-time: "John will send the report by Friday" → auto-extracted as an action item attributed to Speaker "John". Shows as a pinned sidebar list during the meeting, included in summary.

Effort: Medium — builds on existing intelligence pipeline + new UI panel
5

User Highlight / Bookmark Moments

Hotkey (e.g., Ctrl+H) to bookmark the current moment during a meeting — adds a marker in the transcript with optional quick note. In a lecture: "This will be on the exam." In a meeting: "Key decision made here." Bookmarks show in post-meeting review.

Effort: Low — hotkey + timestamp marker + UI indicator
6

Transcription Confidence Indicator

Subtly highlight low-confidence words in the transcript (lighter opacity or dotted underline). In noisy in-person environments, this tells the user which parts might be inaccurate. Clickable to see alternatives or flag for review.

Effort: Low — STT providers already return confidence scores
7

Multi-Format Export

Post-meeting export to Markdown, PDF, SRT (subtitles), or structured JSON. Lecture mode could export as study notes. Team meeting exports as minutes with action items. Different formats for different scenarios.

Effort: Medium — templated export per scenario type
8

Noise Environment Presets

In-person environments vary: quiet office, noisy classroom, conference hall. Presets auto-tune VAD sensitivity, noise gate, and STT parameters. "Classroom" mode expects echo and background chatter; "Quiet Office" mode is more sensitive to soft speech.

Effort: Medium — VAD parameter tuning + preset configs

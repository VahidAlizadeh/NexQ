Speaker Detection Prompt — Redesign

When Deepgram detects a new speaker, this banner appears at the top of the transcript panel

Current Design (Save or Dismiss only)

👤+ New speaker detected: Speaker 3
Problem: No way to reject false detection or merge with existing speaker

Proposed Design — Two Actions

👤
New speaker detected
Identified as Speaker 3 by Deepgram diarization
Name this speaker
Actually this is...
Auto-dismisses in 7s

Behavior

  • Name this speaker — renames speaker_3 to the entered name. All past & future segments with speaker_3 show the new name.
  • "Actually this is..." — merges speaker_3 into the clicked speaker. All segments with speaker_3 get reassigned. speaker_3 is removed from the store.
  • Dismiss (✕ or timeout) — keeps the default name "Speaker 3". No merge, no rename.
  • Timer bar — visual countdown, auto-dismisses after 10s so it doesn't block the UI during fast-paced meetings.

How should users pick the meeting type?

This is the entry point — the first thing users see when starting a meeting. Which pattern feels right for NexQ?

A

Two Distinct Buttons on Dashboard

The launcher always shows two buttons side-by-side: "Online Meeting" and "In-Person Meeting". User picks before anything starts. Clear, zero ambiguity, always visible.

Pros

  • Fastest path — one click
  • Always discoverable
  • No extra modal/step

Cons

  • Takes more dashboard space
  • May feel cluttered if more modes are added later
B

Modal Prompt After "Start Meeting"

Keep the single "Start Meeting" button. After clicking, a modal asks: "What type of meeting?" with Online / In-Person options. Then proceeds to start.

Pros

  • Dashboard stays clean — one primary CTA
  • Scales to more modes easily
  • Good place to show a brief explainer per mode

Cons

  • Extra click every time
  • Users who always use the same mode get annoyed
C

Segmented Toggle + Single Start Button

A small segmented control (pill toggle) on the dashboard: Online | In-Person. It remembers the last selection. The "Start Meeting" button starts whichever mode is active. One glance shows what you'll get.

Pros

  • Remembers preference — power users love this
  • Still one-click to start
  • Compact, doesn't clutter dashboard
  • Clear visual state of what mode is selected

Cons

  • Toggle could be missed by new users
  • Risk of starting wrong mode if toggle is forgotten
D

Smart Default + Quick Switch

App detects likely mode (e.g., if system audio device is active → Online, if only mic → In-Person). Shows a subtle indicator: "Starting as Online Meeting ▾" that's clickable to switch. Combines intelligence with manual override.

Pros

  • Zero friction for the common case
  • Feels smart and premium
  • Still overridable

Cons

  • Auto-detection could be wrong → wrong meeting type started
  • More complex to implement
  • "Magic" can feel unpredictable

Bookmarking a Transcript Line

How should the user interact with a transcript line to bookmark it or add a note?

A

Hover Action Icons

Small icons appear on hover at the right edge of each line: bookmark icon + note icon. Click bookmark to toggle, click note to open inline input below the line.

2:14 Prof Smith
We need to revisit the timeline for Q3 deliverables.
2:31 Speaker 2
The client pushed back on the original deadline, so we have two extra weeks.
2:45 Prof Smith 🔖
Let's lock that in — two weeks extension, final delivery March 28th.
📝 Key decision: deadline extended to March 28

Pro: Discoverable, clean. Con: Hover doesn't work on touch.

B

Click Line → Context Menu

Clicking any transcript line shows a small floating menu with: Bookmark, Add Note, Copy Text. Quick and contextual.

2:14 Prof Smith
We need to revisit the timeline for Q3 deliverables.
2:31 Speaker 2
The client pushed back on the original deadline, so we have two extra weeks.

Pro: Extensible (can add more actions later), works same on live + past meetings. Con: Extra click to get to action.

C

Hover Icons + Right-Click Context Menu

Hover shows bookmark icon (quick toggle). Right-click (or long press) shows full context menu with note, copy, etc. Best of both: fast one-click bookmark, rich menu when needed.

2:31 Speaker 2
The client pushed back on the original deadline.
↑ hover = bookmark icon   |   right-click = full menu ↓
2:45 Prof Smith
Let's lock that in — final delivery March 28th.

Pro: Fastest bookmark (one click), extensible menu, works live + past. Con: Right-click is non-obvious, hover requires mouse.

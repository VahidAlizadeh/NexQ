## Design Context

### Users
Professionals using NexQ during high-stakes meetings and interviews. They need real-time AI assistance (suggested answers, transcription, context retrieval) while staying focused on the conversation. The interface competes for attention with the meeting itself — it must be glanceable, not demanding. Desktop-only (Windows), two-window architecture: a launcher/dashboard (900×650) and an always-on-top overlay (500×700, transparent).

### Brand Personality
**Bold. Refined. Alive.**

NexQ should feel like a luxury instrument — every detail considered, nothing accidental. It evokes the confidence of having a superpower during high-pressure conversations. The interface communicates competence through restraint and precision, not through volume or visual noise.

### Emotional Goals
- **Premium & polished**: Like a well-crafted tool that feels expensive. Users should notice the quality of transitions, the precision of spacing, the thoughtfulness of every micro-interaction.
- **Confidence**: The UI should make users feel like they have an unfair advantage — calm assurance that everything is handled.

### Aesthetic Direction
**Visual tone**: Bold modern minimalism with intentional motion. Dark-first, warm-tinted neutrals (never pure gray). Strong typographic hierarchy. Generous whitespace punctuated by confident color accents.

**References**:
- **Notion**: Clean minimalism, excellent typography, elegant but deeply functional. The way content breathes.
- **Arc Browser**: Bold spatial design, vibrant but controlled color, opinionated layout choices. Things feel alive and intentional.

**Anti-references** (ALL of these are rejected):
- Generic SaaS dashboards (card-grid-metric-chart templates)
- AI-slop aesthetic (purple-blue gradients, cyan-on-dark neon, "futuristic" cliches)
- Cluttered IDE density (VS Code-style overwhelming information)
- Flat & lifeless (plain white cards on gray, "safe and boring")

**Theme**: Dark mode primary, light mode secondary. Both use warm-tinted neutrals (slate-blue family), never pure black/white/gray.

### Design Principles

1. **Every pixel is intentional** — No default values. Spacing, color, typography, and motion all serve a purpose. If you can't explain why something looks the way it does, redesign it.

2. **Motion earns attention** — Animation is not decoration. Every transition communicates state, guides focus, or provides feedback. Orchestrated entrances > scattered micro-interactions. Spring physics for natural deceleration. Never bounce or elastic curves.

3. **Information through hierarchy, not density** — Use size, weight, color, and space to create clear visual layers. The most important element should be obvious within 1 second. Dense data is organized, not crammed.

4. **Color with meaning** — Every color communicates something: blue=primary action, green=success/live, amber=warning/interviewer, red=destructive. Decorative color is banned. Neutral tints are warm (slate-blue family), never pure gray.

5. **Restraint is confidence** — A premium interface trusts the user. No redundant labels, no over-explained states, no visual hedging. Bold choices executed with precision beat safe choices executed generically.

### Technical Constraints
- Tauri 2 + WebView2 (Chromium-based, modern CSS fully supported)
- React 18 + Tailwind CSS 3.4 + shadcn/ui components
- Two-window rendering: launcher + overlay (both render the same React app)
- Performance: 60fps animations mandatory, GPU-accelerated transforms only
- Accessibility: `prefers-reduced-motion` disables all animation (already implemented)
- Typography: Inter (body), Cascadia Code (mono) — consider upgrading if personality demands it
